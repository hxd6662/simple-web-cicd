# CI/CD 实验项目安装和部署指南

## 1. 安装 Git

### 方法一：手动安装 Git
1. 访问 Git 官方网站：https://git-scm.com/downloads
2. 下载 Windows 版本的 Git 安装程序
3. 运行安装程序，按照默认选项安装
4. 安装完成后，重启 PowerShell 或命令提示符

### 方法二：使用 Scoop 安装（如果已安装 Scoop）
```powershell
scoop install git
```

## 2. 配置 Git

安装完成后，打开 PowerShell 并运行以下命令配置 Git：

```powershell
git config --global user.name "你的姓名"
git config --global user.email "你的邮箱@example.com"
```

## 3. 创建 GitHub 仓库

1. 访问 https://github.com 并登录你的账户
2. 点击右上角的 "+" 按钮，选择 "New repository"
3. 输入仓库名称：`simple-web-cicd`
4. 选择 "Public" 或 "Private"
5. 不要勾选 "Add a README file"（因为我们已经有了）
6. 点击 "Create repository"

## 4. 初始化 Git 仓库并推送代码

在项目目录中打开 PowerShell，运行以下命令：

```powershell
# 初始化 Git 仓库
git init

# 添加所有文件到暂存区
git add .

# 提交代码
git commit -m "初始提交：添加学号和姓名信息"

# 添加远程仓库（替换 YOUR_USERNAME 为你的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USERNAME/simple-web-cicd.git

# 推送代码到 GitHub
git push -u origin main
```

## 5. 设置 GitHub Secrets

为了 CI/CD 流水线能够部署到你的云服务器，需要设置以下 Secrets：

1. 在 GitHub 仓库页面，点击 "Settings"
2. 在左侧菜单中选择 "Secrets and variables" → "Actions"
3. 点击 "New repository secret" 添加以下 Secret：

### SSH_HOST
- 名称：`SSH_HOST`
- 值：你的云服务器公网 IP 地址

### SSH_USERNAME
- 名称：`SSH_USERNAME`
- 值：`root`（或你的服务器用户名）

### SSH_PRIVATE_KEY
- 名称：`SSH_PRIVATE_KEY`
- 值：你的 SSH 私钥内容（整个私钥文件内容）

## 6. 准备云服务器

### 6.1 申请云服务器
1. 访问阿里云、腾讯云或华为云
2. 申请免费的云服务器 ECS（确保带有 Docker）
3. 记录服务器的公网 IP 地址

### 6.2 配置服务器
1. 通过 SSH 连接到你的服务器
2. 确保 Docker 已安装并运行：
   ```bash
   docker --version
   docker compose --version
   ```
3. 创建部署目录：
   ```bash
   mkdir -p /opt/simple-web
   ```

## 7. 运行 CI/CD 流水线

1. 在 GitHub 仓库页面，点击 "Actions" 标签
2. 你会看到 "Flask CI/CD" 工作流
3. 点击 "Run workflow" 按钮（或直接推送代码触发）
4. 工作流将自动运行：
   - ① 验证 & 测试
   - ② 构建镜像
   - ③ 部署至 ECS
   - ④ 结果通知

## 8. 访问部署的应用

部署成功后，你可以通过以下方式访问：

1. 浏览器访问：`http://你的服务器IP`
2. 健康检查：`http://你的服务器IP/health`

## 9. 更新到 2.0 版本

1. 修改 `app.py` 中的版本号（将 `v1.0` 改为 `v2.0`）
2. 提交并推送代码：
   ```powershell
   git add .
   git commit -m "更新到 2.0 版本"
   git push origin main
   ```
3. GitHub Actions 会自动重新部署

## 10. 回退到上一个版本

如果需要回退到上一个版本，可以使用 Git 命令：

```powershell
# 查看提交历史
git log --oneline

# 回退到上一个版本（保留更改）
git reset --soft HEAD~1

# 或者回退到上一个版本（丢弃更改）
git reset --hard HEAD~1

# 强制推送（谨慎使用）
git push -f origin main
```

## 故障排除

### 1. Git 命令不可用
- 确保 Git 已正确安装并添加到 PATH 环境变量
- 重启 PowerShell 或命令提示符

### 2. GitHub Actions 失败
- 检查 GitHub Secrets 是否正确设置
- 查看 Actions 日志获取详细错误信息
- 确保云服务器可以正常访问

### 3. 部署失败
- 检查云服务器的 Docker 是否正常运行
- 确保服务器端口 80 未被占用
- 查看服务器日志：`docker compose logs`
